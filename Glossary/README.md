# GLOSSARY - THUẬT NGỮ DỰ ÁN E-OFFICE HÒA PHÁT

Tài liệu này định nghĩa chi tiết các thuật ngữ, từ viết tắt và các tác nhân tham gia vào hệ thống Quản lý Văn phòng – Hành chính của Tập đoàn Hòa Phát.
| Thuật ngữ | Định nghĩa |
| :--- | :--- |
| **Actor** | Tác nhân: Người dùng hoặc hệ thống bên ngoài có tương tác trực tiếp với hệ thống. |
| **User** | Người dùng: Bất kỳ cá nhân nào được cấp tài khoản để truy cập và sử dụng hệ thống. |
| **Nhân viên** | Người dùng thông thường có quyền tạo hồ sơ, gửi đề xuất, tra cứu tài liệu và theo dõi trạng thái. |
| **Trưởng phòng** | Quản lý cấp phòng ban, có quyền xem và phê duyệt hồ sơ thuộc phạm vi đơn vị phụ trách. |
| **Ban lãnh đạo** | Nhóm người dùng cấp quản lý cao, có quyền phê duyệt hồ sơ quan trọng và xem báo cáo tổng hợp. |
| **Văn thư** | Bộ phận chịu trách nhiệm tiếp nhận, vào sổ, lưu trữ, phân phối và quản lý công văn. |
| **Admin** | Quản trị viên hệ thống: Quản lý tài khoản, phân quyền, cấu hình và hỗ trợ vận hành. |
| **Role** | Vai trò: Nhóm quyền hạn được gán cho người dùng (Ví dụ: Nhân viên, Lãnh đạo, Admin). |
| **Phòng ban** | Đơn vị chức năng như Hành chính, Nhân sự, Kế toán, CNTT, Sản xuất, Kinh doanh. |
| **Chi nhánh / Nhà máy** | Đơn vị trực thuộc hoạt động tại các địa điểm khác nhau nhưng dùng chung hệ thống. |
| **Công văn đến** | Văn bản, thông báo gửi từ bên ngoài đến công ty được tiếp nhận xử lý trên hệ thống. |
| **Công văn đi** | Văn bản do doanh nghiệp phát hành ra bên ngoài sau khi phê duyệt nội bộ. |
| **Hồ sơ nội bộ** | Các tài liệu, biểu mẫu, quyết định hoặc văn bản phát sinh trong nội bộ công ty. |
| **Phiếu đề xuất** | Biểu mẫu điện tử nhằm đề nghị mua sắm, thanh toán hoặc các nhu cầu hành chính khác. |
| **Workflow** | Quy trình xử lý nghiệp vụ tự động từ lúc khởi tạo đến khi hoàn thành. |
| **Phê duyệt đa cấp** | Cơ chế xử lý hồ sơ qua nhiều cấp quản lý theo đúng cơ cấu tổ chức tập đoàn. |
| **Kho tài liệu số** | Nơi lưu trữ tập trung các tài liệu điện tử, công văn và biểu mẫu của doanh nghiệp. |
| **Lịch họp** | Lịch làm việc và các cuộc họp được quản lý tập trung trên hệ thống. |
| **Đặt phòng họp** | Chức năng cho phép người dùng đăng ký sử dụng phòng họp theo thời gian cụ thể. |
| **Dashboard** | Màn hình tổng hợp hiển thị số liệu, hồ sơ chờ xử lý và các báo cáo quản trị. |
| **Notification** | Thông báo/Cảnh báo khi có hồ sơ mới hoặc thay đổi trạng thái công việc. |
| **RBAC** | Role-Based Access Control: Mô hình phân quyền truy cập dựa trên vai trò của người dùng. |
| **Audit Log** | Nhật ký hệ thống ghi nhận toàn bộ thao tác (đăng nhập, sửa, xóa) để truy vết. |
| **Search Engine** | Công cụ hỗ trợ tìm kiếm nhanh tài liệu theo từ khóa, ngày tạo hoặc trạng thái. |
| **Requirement** | Yêu cầu: Điều kiện hoặc chức năng mà hệ thống phải đáp ứng. |
| **Functional Req** | Yêu cầu chức năng: Các hành động cụ thể hệ thống thực hiện (Tạo công văn, duyệt...). |
| **Non-functional Req**| Yêu cầu phi chức năng: Các tiêu chuẩn về hiệu năng, bảo mật, độ ổn định. |
| **Use Case** | Trường hợp sử dụng: Mô tả tương tác giữa tác nhân và hệ thống để đạt mục tiêu. |
| **Database** | Cơ sở dữ liệu: Hệ thống lưu trữ tập trung toàn bộ dữ liệu của dự án. |
| **Backup** | Sao lưu dữ liệu định kỳ nhằm phòng ngừa mất mát thông tin. |
| **Recovery** | Khôi phục hệ thống: Quá trình phục hồi dữ liệu/dịch vụ sau khi xảy ra sự cố. |
| **Security** | Bảo mật: Các biện pháp bảo vệ dữ liệu khỏi truy cập trái phép hoặc tấn công. |
| **HTTPS** | Giao thức truyền dữ liệu an toàn thông qua mã hóa giữa người dùng và máy chủ. |
| **SSO** | Single Sign-On: Cơ chế đăng nhập một lần để truy cập nhiều hệ thống nội bộ. |
| **OCR** | Công nghệ nhận dạng ký tự từ ảnh scan hoặc file PDF thành văn bản. |
| **AI Classification** | Phân loại bằng AI: Tự động nhận diện loại công văn hoặc nhóm hồ sơ. |
| **AI Recommendation** | Gợi ý bằng AI: Đề xuất tài liệu liên quan hoặc lịch họp phù hợp. |
| **Responsive Design** | Giao diện tự điều chỉnh tương thích với PC, Tablet và Smartphone. |
| **API** | Giao diện kết nối cho phép tích hợp với ERP, HRM, Email hoặc Chữ ký số. |
| **ERP / HRM** | Hệ thống hoạch định nguồn lực và quản lý nhân sự của doanh nghiệp. |
| **Traceability** | Khả năng truy vết mối liên kết giữa yêu cầu, chức năng và lịch sử thao tác. |
| **Vision Document** | Tài liệu mô tả mục tiêu tổng thể và giá trị mà hệ thống mang lại. |
